# remnanode-bash2 — Server Setup Script

Автоматизированная настройка VPN-ноды на базе Ubuntu (Debian-based).  
Запуск — от `root`.

---

## Требования

| Параметр | Значение |
|---|---|
| ОС | Ubuntu 22.04 / 24.04 (Debian-based) |
| Пользователь | `root` |
| Доступ | SSH, интернет |
| Дополнительно | SSH-доступ к исходной ноде (если копируется сертификат) |

---

## Установка

```bash
curl -O https://raw.githubusercontent.com/sasha33396/remnanode-bash2/main/node.sh && bash node.sh
```

Скрипт интерактивен: **все параметры запрашиваются до начала выполнения**.

---

## Входные параметры

| Параметр | Описание |
|---|---|
| `remnanode SECRET_KEY` | Секретный ключ для авторизации remnanode API(Находится в Remnawave Panel. SECRET_KEY у нод универсален) |
| `xray-sni SNI_DOMAIN` | Домен для TLS SNI (например, `example.com`) |
| `xray-sni CF_API_TOKEN` | Cloudflare API token для DNS-01 challenge |
| `Remnanode API IP` | IP Remnawave Panel, которому разрешён доступ к порту `2222` |
| `Metrics IP` | IP сервера с метриками, которому разрешён доступ к портам `9100`, `9200` |
| `Copy cert from existing node?` | `y` — скопировать сертификат с другой ноды группы если в DNS баллансировку добавляется больше 5 нод за 7 дней, `n` — получить новый |
| `IP of existing node` | IP источника сертификата (только если выбрано `y`) |

---

## Что делает скрипт

| Шаг | Действие |
|---|---|
| 1 | `apt update && apt upgrade`, установка базовых пакетов (`mc`, `htop`, `btop`, `iftop`, `curl`, `wget`) |
| 2 | Установка часового пояса `Europe/Moscow` |
| 3 | Установка Docker (`get.docker.com`) |
| 4 | Загрузка модуля `nf_conntrack`, применение sysctl-параметров (BBR, conntrack, буферы TCP) |
| 5 | Установка лимитов файловых дескрипторов: `300000` (PAM + systemd) |
| 6 | Настройка UFW: SSH, порт 443, ограничение доступа к `2222`/`9100`/`9200` по IP; блокировка известных сканер-сетей; запрет исходящего SMTP |
| 7 | Установка и запуск `fail2ban` |
| 8 | Деплой `remnanode` (Docker Compose, `network_mode: host`, порт `2222`) |
| 9 | Установка `node_exporter` v1.8.2 (systemd-сервис, порт `9100`) |
| 10 | Деплой `speedtest-exporter` (Docker Compose, порт `9200`, server ID `32983`) |
| 11 | Настройка `logrotate` для `/var/log/remnanode` (50 МБ, 5 ротаций, compress) |
| Post | Клонирование `xray-sni`, запись `.env`; копирование сертификата с исходной ноды или ожидание выпуска нового через Caddy DNS-01 |

## Открытые порты после установки

| Порт | Протокол | Доступ | Назначение |
|---|---|---|---|
| `22` | TCP | любой | SSH |
| `443` | TCP | любой | VLESS Reality |
| `2222` | TCP | `REMNA_API_IP` | Remnanode API |
| `9100` | TCP | `METRICS_IP` | Node Exporter |
| `9200` | TCP | `METRICS_IP` | Speedtest Exporter |

## Завершение установки xray-sni

Сервис `xray-sni` **не запускается автоматически**. После завершения скрипта:

```bash
cd /root/xray-sni && docker compose up -d
```

**Если выбрано копирование сертификата (`y`):**  
Сертификат скопирован по пути Caddy-хранилища до первого старта. Caddy использует существующий файл.

**Если выбрано получение нового сертификата (`n`):**  
Caddy запросит сертификат через DNS-01 при первом старте. Требуется корректный `CF_API_TOKEN` с правами на зону домена.

## Поведение при ошибках

При сбое любой команды скрипт останавливается и предлагает выбор:

```
[r]etry  [c]ontinue  [a]bort ?
```

| Опция | Действие |
|---|---|
| `r` | Повторить упавшую команду |
| `c` | Продолжить со следующего шага |
| `a` | Прервать выполнение с кодом выхода упавшей команды |

## Ограничения и допущения

- Скрипт не идемпотентен: повторный запуск может создать дублирующиеся записи в `sysctl.conf` и `limits.conf`.
- SSH-доступ к исходной ноде при копировании сертификата должен быть настроен без пароля (ключ) или принят вручную в рамках `StrictHostKeyChecking=no`.
- `speedtest-exporter` использует фиксированный server ID `32983` — при необходимости изменить до запуска в `/root/speedtest-exporter/docker-compose.yml`.
- `node_exporter` версии `1.8.2` скачивается с GitHub напрямую; при недоступности репозитория шаг завершится ошибкой.
- Блоки IP в UFW статичны и зашиты в скрипт — для актуализации списков требуется ручная правка.
